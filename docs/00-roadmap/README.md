# Go 学习路线图

这个仓库按“语法基础 -> 核心抽象 -> 工程能力 -> 性能优化”的顺序组织。

## 第一阶段：把基础语法捡回来

建议顺序：

1. 开发环境与程序结构
2. 变量、常量、类型与零值
3. 数组、切片、Map 与流程控制
4. 函数、闭包、defer、panic、recover

这一阶段的目标不是刷题，而是重新建立 Go 的思维模型：

- Go 的代码组织方式是什么。
- 零值、值传递、切片共享底层数组是什么意思。
- 为什么 Go 喜欢多返回值。
- `defer`、`panic`、`recover` 的边界在哪里。

## 第二阶段：掌握 Go 最重要的抽象

建议顺序：

1. 结构体、方法、接口
2. 包、模块与泛型
3. 错误处理

这一阶段要重点理解：

- Go 没有类，但为什么结构体 + 方法依然足够强。
- 接口为什么强调“行为”而不是“继承”。
- 泛型适合解决什么问题，不适合解决什么问题。
- 为什么 Go 项目里错误处理看起来“啰嗦”，但可维护性反而更高。

## 第三阶段：开始接近工作场景

建议顺序：

1. 常用标准库：io、json、http、time
2. 并发：goroutine、channel、select
3. context 与取消控制
4. 测试、基准测试、竞态检测

这一阶段是从“会写 Go”走向“能在项目里稳定写 Go”。

## 第四阶段：工程意识与性能意识

最后学习：

1. 性能优化与常见陷阱

这个阶段要开始关注：

- 哪些写法会带来多余分配。
- 为什么预分配切片常常值得做。
- `strings.Builder` 为什么比字符串频繁拼接更稳。
- 什么情况下会意外保留大对象引用。

## 每个模块怎么学

每个模块都建议按这个顺序：

1. 先读文档，明确知识点和坑点。
2. 再看 `examples/` 里的代码实现。
3. 直接运行对应测试。
4. 在测试里打断点单步调试。
5. 自己改一个输入，再跑一次验证理解。

## 最小可执行学习节奏

如果你每天只有 30 到 45 分钟，可以这样安排：

- 第 1 周：01 到 04
- 第 2 周：05 到 07
- 第 3 周：08 到 10
- 第 4 周：11 到 12，并把所有示例自己重写一遍

## 常用命令

```bash
go test ./...
go test -v ./examples/02_basics
go test -run TestAccumulator -v ./examples/04_functions
go test -race ./examples/09_concurrency
go test -bench . -benchmem ./examples/12_performance
```
